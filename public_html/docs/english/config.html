MailChimp Plugin

The MailChimp Plugin allows site visitors to subscribe to announcements and other communications. This serves as a one-way mailing list. Unlike the built-in "Mail Users" function, non-members can also subscribe to mailings.

MailChimp Plugin Configuration Options

Main Settings
Option Description
API Key Enter your MailChimp API key here.
Default List Select the default mailing list for subscribers. This list must already be set up for your account at MailChimp. Currently, only the default list can accept subscriptions through this plugin. Note: You must add your API Key first and save your configuration before MailChimp lists will show up.
Subscribe at Registration Select whether new site users can subscribe to the default mailing list during registration. Some options are not recommended but may be appropriate, depending on your site's purpose.
  • The option to subscribe is not available
  • New uses are automatically subscribed (not recommended)
  • New users may optionally subscribe, checkbox is checked (not recommended)
  • New users may optionally subscribe, checkbox is unchecked
Double-opt-in for profile changes? Select "Yes" to require double opt-in when a user subscribes to your list by editing their own profile. (Recommended)
Sync At Login? Select whether user acconts should be automatically synchronized with MailChimp during login. This is important if you choose not to enable Webhooks, and can be a good backup in case some Webhook messages aren't received. It may also increase the response time to user logins.
Webhook Settings
A Webhook allows MailChimp to send a POST request to your server when a user subscribes, unsubscribes, or changes their email address on MailChimp's web site. The MailChimp plugin maintains a table of subscriber IDs to determine how the subscription options should be shown in the block or account settings. Enabling the Webhook is recommended to help keep your site in sync with MailChimp.
Option Description
Webhook Key When you set the Webhook URL in MailChimp you can add a key value to increase security. Your Webhook URL should look like "http://yoursite/mailchimp/hook.php?key=SecretKey", and you would then enter "SecretKey" in this field. If you omit the key then the Webhook will still work but anyone would be able to manipulate your data by sending a specially-crafted POST request. The parameter mane must be "key".
Webhook handles Subscriptions? Select "Yes" to have users who subscribe directly via MailChimp updated in your database. If they are site members and their email address matches, they'll be added to the local database so their account preferences are in sync and they won't be shown the subscription block when they're logged in. Recommended
Webhook handles Removals? Select "Yes" to have users who unsubscribe directly via MailChimp updated in your database. If they are site members and their email address matches, they'll be removed and will again see the subscription block (if used). This also applies to members who's addresses are cleaned automatically by MailChimp. Recommended
Update user email when MailChimp email changes? Select "Yes" to have users who update their email address with MailChimp also have their address changed in your database.
